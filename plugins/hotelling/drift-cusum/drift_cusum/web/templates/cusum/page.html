CUSUM (Cumulative Sum)

누적합 기반 변화점 탐지 알고리즘. 평균의 점진적 이동에 민감하다.

CUSUM 알고리즘

개요

CUSUM(Cumulative Sum)은 시계열 데이터의 평균 변화를 탐지하는 순차 분석 기법이다. 1954년 E.S. Page가 제안했으며, 품질 관리와 프로세스 모니터링에서 널리 사용된다.

원리

관측값이 기준 평균에서 벗어나는 정도를 누적하여, 누적합이 임계값을 초과하면 변화가 발생했다고 판단한다.

  • S+ (양방향): 평균이 상승하는 방향의 누적 이탈. S+(t) = max(0, S+(t-1) + z(t) - k)
  • S- (음방향): 평균이 하락하는 방향의 누적 이탈. S-(t) = max(0, S-(t-1) - z(t) - k)
  • Alarm: S+ > h 또는 S- > h 이면 drift 감지

여기서 z(t)는 관측값을 표준화(median, MAD)한 값이다.

파라미터

파라미터의미기본값효과
k (Slack) 허용 편차. 이 값보다 작은 변화는 무시한다. 0.25 작을수록 민감 (작은 변화도 감지), 클수록 둔감 (큰 변화만 감지)
h (Threshold) 알람 임계값. 누적합이 이 값을 넘으면 alarm. 5.0 작을수록 빠른 감지 (오탐 증가), 클수록 느린 감지 (오탐 감소)
reset alarm 발생 후 누적합을 0으로 초기화할지 여부. True True: 독립적인 이벤트로 분리 / False: 연속적 추적

표준화

CUSUM을 적용하기 전에 원본 데이터를 robust 표준화한다:

  • 위치: median (평균 대신 — 이상치에 강건)
  • 척도: MAD × 1.4826 (표준편차 대신 — 이상치에 강건)
  • z(t) = (x(t) - median) / (MAD × 1.4826)

적합한 상황

  • AI 모델의 confidence score가 서서히 하락하는 경우
  • 센서 데이터의 평균이 점진적으로 이동하는 경우
  • 갑작스러운 변화보다 느린 drift를 감지하고 싶은 경우

한계

  • 분산(산포)의 변화는 감지하지 못한다 (평균 변화만 감지)
  • 다변량 데이터에는 직접 적용할 수 없다 (단변량 전용)
  • k, h 파라미터를 잘못 설정하면 과다 탐지 또는 미탐지 발생

참고 문헌

  • Page, E.S. (1954). "Continuous Inspection Schemes." Biometrika, 41(1/2), 100-115.
  • Montgomery, D.C. (2009). Introduction to Statistical Quality Control, 6th ed. Wiley.

데이터 구성

합성 데이터 300건으로 구성된다. 정상 구간과 drift 구간이 포함되어 있어 CUSUM의 동작을 확인할 수 있다.

구간건수평균표준편차설명
정상 구간 200건 0.90 0.02 AI 모델이 정상적으로 동작하는 상태의 confidence score
Drift 구간 100건 0.80 0.03 모델 성능이 저하되어 confidence가 하락한 상태

시간 설정

  • 시작: 2026-01-01 00:00
  • 간격: 10분
  • 총 기간: 약 2일

기대 결과

  • 정상 구간(1~200번째)에서는 alarm이 거의 발생하지 않아야 한다.
  • drift 구간(201~300번째)에서는 S- (음방향 CUSUM)이 threshold를 초과하여 alarm이 발생해야 한다.
  • 평균이 0.90 → 0.80으로 하락했으므로, S-가 반응하는 것이 정상이다.

차트 읽는 법

  • 상단 차트 (Original Time Series): 원본 데이터. 빨간 점이 alarm이 발생한 시점.
  • 하단 차트 (CUSUM S+/S-): 녹색(S+)은 상승 감지, 빨간색(S-)은 하락 감지. 노란 점선이 threshold(H).
  • S+ 또는 S-가 노란 점선을 넘으면 drift가 감지된 것이다.

파라미터 변경 실험

위 Parameters 영역에서 k, h 값을 바꿔보면 감지 민감도가 변하는 것을 확인할 수 있다:

  • k=0.1, h=3.0: 매우 민감 — 정상 구간에서도 alarm 발생 (오탐 증가)
  • k=0.25, h=5.0: 기본값 — drift 구간에서 적절히 감지
  • k=0.5, h=8.0: 둔감 — drift 구간에서도 alarm이 줄어듦

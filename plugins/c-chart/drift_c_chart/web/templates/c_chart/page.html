C Chart (Count Chart)

결함/사건 건수를 모니터링하는 제어 차트. 포아송분포 기반 제어 한계를 적용한다.

C Chart 알고리즘

개요

C 차트는 일정 단위(시간, 배치 등)에서 발생하는 결함/사건의 건수를 모니터링하는 제어 차트이다. 포아송분포를 기반으로 한다. 각 단위의 결함 건수 c를 기록한다.

제어 한계

  • CL = c̄ (전체 평균 결함 수)
  • UCL = c̄ + 3 × √c̄
  • LCL = max(0, c̄ - 3 × √c̄)

AI 모니터링 적용

시간대별 에러 건수, timeout 건수, alarm 건수 등 카운트 데이터 모니터링. 예를 들어 시간당 추론 에러 건수가 정상 범위를 벗어나면 alarm.

파라미터

파라미터의미기본값효과
reference_ratio 기준 구간 비율. 0.5 기준 구간이 안정적일수록 제어 한계가 정확해진다.

예제 데이터

데이터 구성

합성 데이터: 60개 시간대의 에러 건수.

구간시간대 수평균 건수분포설명
정상 구간 (1~40) 40개 3건/시간 포아송 λ=3 안정적 에러 발생
이상 구간 (41~60) 20개 12건/시간 포아송 λ=12 에러 건수 급증

기대 결과

  • C Chart에서 건수 급증 구간이 UCL을 초과

차트 읽는 법

  • C Chart: 각 시간대의 건수(막대)와 UCL/LCL/CL 라인. 빨간색 막대가 alarm 시점.
  • 건수가 UCL 위로 올라가면 에러 건수가 비정상적으로 많아진 것이다.

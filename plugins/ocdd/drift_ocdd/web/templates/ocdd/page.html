OCDD (One-Class Drift Detector)

기준 분포의 통계량(평균, 표준편차)과 슬라이딩 윈도우를 비교하여 drift를 탐지한다.

OCDD 알고리즘

개요

One-Class Drift Detector는 기준 분포의 rolling statistics(평균, 표준편차)를 학습한 뒤, 슬라이딩 윈도우의 통계량이 기준에서 벗어나는 정도를 z-score로 측정한다.

원리

  • 기준 구간에서 윈도우 평균/표준편차의 분포를 추정
  • 테스트 구간의 각 윈도우에 대해 z-score 계산
  • z_mean: 윈도우 평균의 z-score
  • z_std: 윈도우 표준편차의 z-score
  • 둘 다 임계값을 초과하면 drift (평균 이동 + 분산 변화 동시 감지)
  • Score = max(z_mean, z_std) / z_threshold

파라미터

파라미터의미기본값효과
Window Size 슬라이딩 윈도우 크기 50 클수록 안정적이나 지연 증가
Reference Ratio 기준 구간 비율 0.5 전체 데이터에서 기준 분포로 사용할 비율
Z Threshold Z-score 임계값 3.0 작을수록 민감, 클수록 보수적

적합한 상황

  • 평균 이동과 분산 변화를 동시에 감지하고 싶은 경우
  • 단일 클래스(정상) 학습 기반의 이상 탐지가 필요한 경우
  • 복합적인 분포 변화를 포착하고 싶은 경우

참고 문헌

  • Krawczyk, B. and Wozniak, M. (2015). "One-class classifiers with incremental learning and forgetting for data streams with concept drift." Soft Computing, 19(12), 3387-3400.

차트 읽는 법

  • 상단 차트: 원본 값(검정). 빨간 점이 alarm 시점.
  • 하단 차트: Z-score(녹색: 평균, 빨강: 표준편차). 노란 점선이 임계값.
  • 두 Z-score 모두 임계값을 넘으면 drift가 감지된 것이다.

# X-bar/R Chart

## 개요
데이터를 서브그룹으로 묶어 각 그룹의 평균(X-bar)과 범위(R)를 동시에 모니터링하여 평균 이동과 산포 변화를 감지하는 이중 제어 차트 기반 drift 탐지 알고리즘이다.

## 알고리즘 원리

### 수학적 배경

X-bar/R Chart는 가장 전통적인 변수형 제어 차트로, 서브그룹의 평균과 범위를 별도의 차트로 관리한다.

**서브그룹 통계량:**

각 서브그룹 $j$ (크기 $n$)에 대해:
- 평균: $\bar{x}_j = \frac{1}{n}\sum_{i=1}^{n} x_{ji}$
- 범위: $R_j = \max(x_{ji}) - \min(x_{ji})$

**기준 구간 통계량:**
- 총 평균: $\bar{\bar{x}} = \frac{1}{m}\sum_{j=1}^{m} \bar{x}_j$
- 평균 범위: $\bar{R} = \frac{1}{m}\sum_{j=1}^{m} R_j$

**X-bar 차트 제어 한계:**

$$UCL_{\bar{x}} = \bar{\bar{x}} + A_2 \cdot \bar{R}$$

$$LCL_{\bar{x}} = \bar{\bar{x}} - A_2 \cdot \bar{R}$$

$$CL_{\bar{x}} = \bar{\bar{x}}$$

**R 차트 제어 한계:**

$$UCL_R = D_4 \cdot \bar{R}$$

$$LCL_R = D_3 \cdot \bar{R}$$

$$CL_R = \bar{R}$$

**제어 차트 상수 (서브그룹 크기 n=5 기준):**

| 상수 | 값 | 의미 |
|---|---|---|
| $A_2$ | 0.577 | X-bar 차트 제어 한계 계수 |
| $D_4$ | 2.114 | R 차트 상한 제어 한계 계수 |
| $D_3$ | 0.0 | R 차트 하한 제어 한계 계수 |

**알람 조건:**

$$\bar{x}_j > UCL_{\bar{x}} \; \text{또는} \; \bar{x}_j < LCL_{\bar{x}} \; \text{또는} \; R_j > UCL_R \; \text{또는} \; R_j < LCL_R$$

**Score:**

$$\text{score} = \frac{\max|\bar{x}_j - \bar{\bar{x}}|}{A_2 \cdot \bar{R}}$$

### 핵심 아이디어

X-bar 차트는 프로세스 평균의 변화를, R 차트는 프로세스 산포의 변화를 감지한다. 두 차트를 병행 사용함으로써 평균과 분산의 변화를 동시에 모니터링할 수 있다. 서브그룹 평균은 중심극한정리에 의해 개별 관측값보다 정규분포에 가까워지므로, 비정규 데이터에도 상대적으로 견고하다.

## 파라미터

| 파라미터 | 기본값 | 설명 |
|---|---|---|
| `subgroup_size` | 5 | 서브그룹의 크기. 제어 차트 상수에 영향을 준다. |
| `reference_ratio` | 0.5 | 전체 서브그룹에서 기준 구간이 차지하는 비율 |

## 탐지 로직

`detect()` 메서드의 동작 순서:

1. **서브그룹 생성**: 데이터를 `subgroup_size` 크기로 순서대로 분할한다 (나머지는 버림).
2. **그룹 통계량 계산**: 각 서브그룹의 평균($\bar{x}$)과 범위($R$)를 계산한다.
3. **기준 구간 설정**: 서브그룹의 앞쪽 `reference_ratio` 비율을 기준 구간으로 사용한다.
4. **기준 통계량 계산**: $\bar{\bar{x}}$와 $\bar{R}$을 계산한다.
5. **제어 한계 계산**: $A_2=0.577$, $D_4=2.114$, $D_3=0.0$ 상수를 사용하여 X-bar, R 차트의 제어 한계를 계산한다.
6. **알람 판정**: X-bar 또는 R 차트에서 제어 한계를 벗어난 서브그룹을 alarm으로 표시한다.
7. **연속 알람 그룹화**: 인접한 알람 서브그룹 인덱스를 gap=3 기준으로 그룹화한다.
8. **이벤트 생성**: 각 그룹에서 X-bar가 가장 크게 벗어난 서브그룹을 peak로 선택하고, score를 계산한다. 데이터 인덱스를 서브그룹 인덱스에서 역산한다.
9. **심각도 판정**: score >= 2.0이면 critical, >= 1.0이면 warning, 그 외 normal.

## 차트 시각화

- **기본 차트**: X-bar 시계열 + drift 알람 마커
- **전문가 차트**:
  - Y축(좌): Value (X-bar)
    - `UCL (X)` (주황색 수평선): X-bar 상한 제어 한계
    - `LCL (X)` (주황색 수평선): X-bar 하한 제어 한계
  - Y2축(우): Range
    - `Range` (보라색 선): 각 서브그룹의 범위 값
    - `UCL (R)` (빨간색 수평선): R 차트 상한 제어 한계

## 적합한 상황

### 효과적인 경우
- 연속형 데이터를 자연스러운 서브그룹(배치, 시간대 등)으로 묶을 수 있을 때
- 평균과 산포의 변화를 동시에 감시해야 할 때
- 서브그룹 크기가 2~10 정도로 작을 때 (특히 n=5가 전통적)
- 제조 공정, 측정 시스템 등 전통적인 SPC 적용 환경

### 한계점
- 서브그룹 크기가 고정되어 있어, 가변 크기 데이터에는 부적합하다
- 현재 구현은 $n=5$ 기준의 상수를 하드코딩하므로, 다른 서브그룹 크기에서는 상수를 조정해야 한다
- 서브그룹 크기가 10 이상이면 R 차트 대신 S 차트를 사용하는 것이 효율적이다
- 데이터가 서브그룹으로 나누기에 충분하지 않으면 사용할 수 없다
